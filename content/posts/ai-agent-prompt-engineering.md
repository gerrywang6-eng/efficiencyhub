## 什么是提示工程

### 核心定义

**提示工程（Prompt Engineering）** 是设计和优化输入指令（Prompt），让AI Agent产生最佳输出的技术和方法论。

简单说：**学会和AI"说话"的艺术**。

### 为什么提示工程如此重要

**数据说话**：
- 优化提示词可提升输出质量 **200-500%**
- 减少迭代次数 **60%**
- 降低API成本 **40%**

**真实案例**：
> 某企业使用OpenClaw处理客户邮件，原始提示词准确率65%，优化后达到92%，每月节省人工处理时间120小时。

### 提示工程的三个层次

**Level 1：基础提示**
- 直接表达需求
- 简单明了
- 适合简单任务

**Level 2：结构化提示**
- 使用框架模板
- 提供上下文
- 明确输出格式

**Level 3：高级优化**
- 多轮对话设计
- 思维链引导
- 动态提示调整

## 提示工程基础原理

### AI Agent如何处理提示

```
用户输入 → 语义理解 → 上下文分析 → 任务分解 → 执行 → 输出
```

**关键影响因素**：
1. **清晰度**：指令是否明确无歧义
2. **完整性**：是否提供足够上下文
3. **结构化**：是否有清晰的格式
4. **示例**：是否提供参考样本

### 提示的五大要素

**1. 角色定义（Role）**
```
❌ 差：帮我写一篇文章
✅ 好：你是一位资深科技博主，擅长用通俗易懂的语言解释复杂技术概念。请写一篇关于AI Agent的科普文章。
```

**2. 任务描述（Task）**
```
❌ 差：分析这个数据
✅ 好：分析附件中的销售数据，找出过去6个月的增长趋势、季节性模式和异常值。
```

**3. 上下文信息（Context）**
```
❌ 差：优化这段代码
✅ 好：优化这段Python代码。背景：处理10万条数据，当前运行时间30秒，目标是降到5秒以内。内存限制1GB。
```

**4. 输出格式（Format）**
```
❌ 差：列出要点
✅ 好：以Markdown格式输出，包含：1）问题概述 2）3个核心建议 3）实施步骤 4）预期效果
```

**5. 约束条件（Constraints）**
```
❌ 差：写一个方案
✅ 好：写一个营销方案，要求：预算不超过1万元、执行周期2周、适合小型团队、可量化效果。
```

## 高级提示技巧

### 技巧1：Chain-of-Thought（思维链）

**原理**：引导AI分步骤思考，提高复杂任务准确率。

**示例**：
```
问题：一个项目原计划10天完成，前3天完成了30%，之后效率提升20%，问实际几天完成？

请按以下步骤思考：
1. 计算原计划每天完成量
2. 计算前3天实际完成量
3. 计算剩余工作量
4. 计算效率提升后的日完成量
5. 计算剩余工作所需天数
6. 得出总天数
```

### 技巧2：Few-Shot Learning（少样本学习）

**原理**：提供示例，让AI学习模式。

**示例**：
```
将以下产品描述转换为卖点：

示例1：
输入：这款笔记本电脑重1.2kg，电池续航12小时
输出：轻薄便携，全天续航无忧

示例2：
输入：该软件支持100+种文件格式
输出：万能兼容，一站式解决文件转换

现在转换：
输入：[你的产品描述]
输出：
```

### 技巧3：Self-Consistency（自一致性）

**原理**：多次生成，选择最佳答案。

**示例**：
```
请生成3个不同版本的[内容]，然后分析每个版本的优缺点，最后推荐最佳版本并说明理由。
```

### 技巧4：Role-Playing（角色扮演）

**原理**：让AI扮演特定角色，获得专业视角。

**示例**：
```
请分别扮演以下角色，对同一个问题给出建议：
1. 风险投资家（关注回报和风险）
2. 技术专家（关注可行性和创新）
3. 市场营销总监（关注用户和市场）
4. 运营总监（关注执行和成本）

问题：[你的问题]
```

### 技巧5：Iterative Refinement（迭代优化）

**原理**：多轮对话逐步完善输出。

**对话流程**：
```
Round 1：生成初稿
Round 2：指出问题，要求改进
Round 3：针对特定方面深化
Round 4：最终润色
```

## 常见错误与解决方案

### 错误1：提示太模糊

**症状**：AI输出偏离预期
**示例**：
```
❌ 差：写个方案
✅ 好：写一个针对25-35岁职场新人的在线课程营销方案，预算5万元，周期1个月，重点渠道是小红书和微信。
```

### 错误2：上下文不足

**症状**：AI缺乏背景信息，输出不相关
**示例**：
```
❌ 差：优化这段代码
✅ 好：优化这段Python数据处理代码。当前处理10万行CSV需要30秒，目标是5秒内完成。内存限制2GB，可以使用pandas和numpy。
```

### 错误3：约束不明确

**症状**：输出超出预期范围
**示例**：
```
❌ 差：写一篇文章
✅ 好：写一篇800-1000字的科技评论文章，主题是AI Agent，目标读者是技术人员，要求包含3个实际案例，避免过于技术化的术语。
```

### 错误4：角色不清晰

**症状**：输出风格不一致
**示例**：
```
❌ 差：解释区块链
✅ 好：你是一位给10岁孩子讲课的科学老师，请用简单有趣的语言解释什么是区块链，使用类比和例子。
```

### 错误5：输出格式未指定

**症状**：格式不符合要求
**示例**：
```
❌ 差：列出要点
✅ 好：以Markdown格式输出，使用二级标题和 bullet points，每个要点包含：问题、解决方案、预期效果。
```

## 提示工程最佳实践

### 黄金法则

**1. 先思考，再提示**
- 明确你想要什么
- 考虑AI需要什么信息
- 预判可能的误解

**2. 从简单开始**
- 先用基础提示测试
- 根据输出逐步优化
- 不要一开始就追求完美

**3. 迭代优化**
- 保存有效的提示模板
- 记录每次修改和效果
- 建立个人提示库

**4. 测试多样性**
- 用不同角度测试同一提示
- 验证输出的稳定性
- 准备备选方案

### 提示优化检查清单

- [ ] 角色是否明确？
- [ ] 任务是否具体？
- [ ] 上下文是否完整？
- [ ] 格式是否指定？
- [ ] 约束是否清晰？
- [ ] 示例是否提供？
- [ ] 输出长度是否合适？
- [ ] 语气是否恰当？

### 建立个人提示库

**建议结构**：

prompts/
